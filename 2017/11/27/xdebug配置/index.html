xdebug在lnmp中的配置

发表于 2017-11-27

在我使用过的工具中，PHP的调试工具Xdebug具有着看似配置简单，实则全是玄学的特点。尽管互联网上关于Xdebug的配置都有着详尽的说明，不过各种眉眉角角真的非常折腾人，各种说明之间相互补充又各有矛盾之处。如果是本地wamp的简单环境配置相信不会太困难，本文主要针对vagrant+虚拟机下的lnmp环境的配置加以说明。

1.下载&编译

欸，下载要说明么。我觉得唯一需要注意的点就是要对应PHP 的版本。
官网教程：https://xdebug.org/docs/install

wget 所需安装包并解压，进入xdebug目录进行
/usr/local/php/bin/phpize && ./configure --with-php-config=/usr/local/php/bin/php-co nfig && make && make install 编译三连

2.php配置
找到有效的php.ini，增加以下配置
zend_extension="/usr/local/php/modules/xdebug.so"
这个时候phpinfo之中应该就已经有了xdebug 的支持。但仍需要更详细的配置。

xdebug.auto_trace=1 xdebug.collect_params=1 xdebug.collect_return=1 xdebug.trace_output_dir="/tmp/xdebug/trace" xdebug.profiler_enable=1 xdebug.profiler_output_dir="/tmp/xdebug/profiler" xdebug.remote_enable=true xdebug.remote_host=10.0.2.15 xdebug.remote_port=9003 xdebug.remote_handler=dbgp xdebug.remote_connect_back=1 xdebug.remote_autostart=1 xdebug.idekey=PHPSTORM

其中因人而异的是remote_host和remote_port。vagrant所配置的虚拟机与外界联系的ip地址一般是192开头的，而此处应该填写10开头的ip
可通过ifconfig查看。
port则是自己与host机上的IDE约定好，一般为9000。
IDEKEY则根据你的IDE或者调试工具进行配置。

3.PHPSTORM配置
PHPSTORM在xdbeug的支持上非常优秀，配置上有些眉角需要注意。

Xdebug配置选项中设置与php.ini中的端口号。

配置所需要监听的servers，一般是你接口的请求域名，端口默认80，这里需要注意的是，一定要勾选use path mapping.在下面把本地文件地址和服务器文件地址对应起来，一般来说只需要配置根目录就够了。

选择run->Edit Configurations

这里很关键哦，如果你希望只要请求就开启调试，那么请选择remote debug，并且选择之前配好的服务，和idekey，配置方面大功告成!
